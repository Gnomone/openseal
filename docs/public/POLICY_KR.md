# 🧶 OpenSeal: 설계 원칙 및 공개 정책 (Protocol Strategy)

본 문서는 OpenSeal v2.0의 설계 철학, 위협 모델, 그리고 전략적 공개 정책을 정의합니다.

---

## 1. 설계 원칙 (Design Rationale)

OpenSeal은 **"실행이 곧 증명이다(Execution is Proof)"**라는 원칙 위에 세워졌습니다. AI 에이전트와 자동화된 인프라 시대에, 중앙 대행 기관 없이 API 응답의 무결성을 검증하는 것은 신뢰의 핵심 연결 고리입니다.

### 왜 OpenSeal인가?
- **범용적 무결성**: 특수 하드웨어가 필요한 TEE나 연산 비용이 막대한 ZK 방식과 달리, OpenSeal은 모든 언어와 OS에서 작동하는 **실용적인 무결성 계층**을 제공합니다.
- **인프라가 아닌 프로토콜**: OpenSeal은 단순한 런타임이 아닙니다. 결과값을 특정 코드 정체성에 고정하는 **표준 프로토콜**입니다.
- **전략적 락인**: 로직과 결과의 바인딩에 집중함으로써, 도입은 쉽지만 운영상의 보안과 생태계 결합도는 흉내 내기 어려운 표준을 지향합니다.

---

## 2. 위협 모델 및 보장 (Threat Model & Guarantees)

OpenSeal이 명시적으로 보호하는 범위와 한계를 정의하여 "정직한 실행" 모델을 보장합니다.

### ✅ 보장하는 것 (Guarantees)
- **로직 무결성**: 검증자는 결과값이 공개적으로 감사된 **정확히 동일한 소스 코드(A-hash)**에 의해 생성되었음을 100% 확신할 수 있습니다.
- **신선함 (재전송 방지)**: 모든 응답은 일회성 챌린지(Wax)에 묶여 있어, 공격자가 과거의 유효한 응답을 재사용하는 것이 불가능합니다.
- **샌드박스 격리**: 앱은 OpenSeal 부모 프로세스가 모든 입출력을 가로채는 정제된 환경에서 실행되어, 조용히 조작되는 것을 방지합니다.

### ❌ 보장하지 않는 것 (Out of Scope)
- **호스트 메모리 보안**: 실행 중 메모리 후킹을 시도하는 물리적 권한의 공격을 방어하지 않습니다. 이는 TEE의 영역입니다.
- **외부 데이터의 진실성**: OpenSeal은 **로직이 정직하게 실행됨**을 증명할 뿐, 해당 로직이 가져온 외부 데이터(예: 외부 거래소 시세) 자체가 객관적으로 참임을 보장하지는 않습니다.

---

## 3. 계층적 공개 모델 (A-E Layers)

OpenSeal은 투명성과 보안의 균형을 위해 코드를 5개의 레이어로 분리하여 관리합니다.

| 레이어 | 컴포넌트 | 공개 여부 | 이유 |
| :--- | :--- | :--- | :--- |
| **Layer A** | UX / CLI | ✅ **공개 (OSS)** | 도입 가속화 및 사용자 신뢰 확보 |
| **Layer B** | 정체성 로직 | ✅ **공개 (OSS)** | 코드 해싱 방식의 투명한 공개 |
| **Layer C** | 런타임 중계 | ❌ **비공개 / 인터페이스** | 실행 환경 우회 공격 방지 |
| **Layer D** | 인감 생성 핵심 | ❌ **절대 비공개** | 프로젝트의 핵심 "경제적 해자" 보호 |
| **Layer E** | 검증기 (Verifier) | ✅ **공개 (OSS)** | 누구나 검증 가능한 신뢰의 앵커 |

---

## 4. 신뢰 모델 (Trust Model)

### 플랫폼은 발견을, 암호학은 신뢰를

OpenSeal의 신뢰는 중앙 플랫폼이 아닌 **수학적 검증**에서 나옵니다.

| 역할 | 플랫폼(HighStation) | OpenSeal 암호학 |
| :--- | :--- | :--- |
| **공급자 발견** | ✅ 플랫폼이 제공 | - |
| **평판 관리** | ✅ 플랫폼이 제공 | - |
| **결제 중개** | ✅ 플랫폼이 제공 | - |
| **결과 검증** | ❌ 플랫폼 불필요 | ✅ 누구나 독립적으로 검증 |

### 누구나, 어디서나, 언제든지

- **공급자**: 배포 전 자가 테스트
- **소비자**: API 응답의 무결성 확인
- **감사자**: 제3자 독립 검증
- **플랫폼**: 평판 점수 계산용 검증

모두 동일한 오픈소스 검증기를 사용하며, 중앙 권한 없이 작동합니다.

---

## 5. 개인정보 보호 및 무결성 공약

OpenSeal 보호된 런타임은 **제로 백엔드(Zero-Backend) 정책**을 기반으로 구축되었습니다.
- **데이터 수집 제로**: 사용자 데이터, 실행 결과 혹은 메타데이터를 절대 수집하거나 저장하지 않습니다.
- **외부 통신 제로**: 핵심 생성 엔진은 완전한 격리 상태에서 작동하며, 인가되지 않은 외부 네트워크 호출을 수행하지 않습니다.
- **감사 가능한 경계**: 오픈소스 CLI와 보호된 런타임 사이의 인터페이스 경계(ABI/IPC)는 명확하게 정의되어 감사가 가능합니다.

---

## 5. 최종 선언
> **"누구나 증명을 검증할 수 있지만, 증명 생성 엔진의 보안은 보호된 경계에 의해 보장됩니다."**

OpenSeal은 학술적 투명성보다 **생태계의 실질적 보안**을 우선시합니다. 우리는 수학과 운영의 일관성을 통해 신뢰를 제공합니다.
