# 📜 OpenSeal v2.0 프로젝트 변경 이력 (Version History)

이 문서는 OpenSeal 프로젝트의 각 버전별 주요 변경 사항, 보안 하드닝 및 기능 추가 내역을 기록합니다.

---

## 🚀 최신 릴리즈 (Current)

### [v2.0-rc29] - 2026-01-16
*   **루프백 통신 최적화**: 프로젝트 전반의 `localhost`를 명시적 IP인 `127.0.0.1`로 교체하여 DNS 의존성을 제거하고 보안 고립성(Isolation)을 강화함.
*   **문서 동기화**: 퀵스타트 및 테스트 스크립트의 통신 주소를 `127.0.0.1`로 일괄 업데이트.

### [v2.0-rc28] - 2026-01-16
*   **Lazy Init (자동 설정 생성)**: `openseal build` 실행 시 `.opensealignore` 및 `.openseal_mutable` 파일이 없으면 템플릿과 함께 자동 생성하는 기능 추가.
*   **코드 클린업**: 컴파일 시 발생하던 미사용 변수 및 임포트 경고(Unused warnings) 전수 제거.

### [v2.0-rc27] - 2026-01-16
*   **파이썬 환경 가이드 보완**: 제외 규칙 예시에 `venv/`를 추가하여 가상 환경 관리에 대한 명확한 예시 제공 (한/영 README 공통).

### [v2.0-rc26] - 2026-01-16
*   **제외 규칙 동작 기준 상세화**: `.opensealignore`가 해당 파일 위치(디렉토리)를 기준으로 상대 경로가 적용됨을 명시.
*   **Self-Inclusion 원칙**: 설정 파일 자체도 '파일 비수정 검사' 대상에 포함되어 규칙 변조를 통한 보안 우회가 불가능함을 안내.

## 🛠️ 주요 마일스톤 (Past Milestones)

### [v2.0-rc21 ~ v2.0-rc25] - 2026-01-15
*   **용어 정립**: 'A-hash'를 직관적인 **'파일 비수정 검사(File Integrity Check)'**로 명칭 변경 및 모든 문서 반영.
*   **`.opensealignore` 도입**: Git 규칙과 별개로 OpenSeal 봉인 단계에서만 파일/폴더를 제외할 수 있는 전용 무시 파일 지원 추가.
*   **다국어 지원**: 한국어 및 영어 README/가이드라인 완전 동기화.

### [v2.0-rc15 ~ v2.0-rc20] - 2026-01-15
*   **OSIP-7325 프로토콜 확립**: 3-Phase(Onboarding -> Challenge -> Verification) 운영 모델 공식 문서화.
*   **보안 아키텍처 강화**: `g_B` 로직의 비공개 원칙(Black-box) 및 격리된 런타임(Isolated Runtime) 가시화.
*   **Server-side Verification**: 검증자가 직접 클론하여 `RootHash`를 추출하는 프로세스 정립.

### [v2.0-rc01 ~ v2.0-rc14] - 2026-01-14
*   **v2.0 아키텍처 전환**: WASM 탈피 및 암호화된 네이티브 런타임 도입.
*   **머클 트리 봉인**: 단일 파일이 아닌 프로젝트 전체 소스코드의 무결성을 보장하는 구조 구현.
*   **독점적 호출(Caller Monopoly)**: 부모 프로세스가 자식 프로세스(앱)를 격리/제어하는 러너 모델 구축.
*   **환경 소독**: `env_clear()` 및 환경변수 화이트리스트 필터링 구현.

## 📅 향후 계획 (Roadmap)
*   **HighStation SDK**: 고수준 언어(Rust/JS/Go)용 검증 SDK 개발.
*   **Secret Salt 주입**: `g_B` 로직에 검증자만 아는 비밀값(Salt)을 동적으로 결합하는 보안 고도화.
*   **CLI 난독화**: 봉인 생성 로직이 포함된 CLI 바이너리의 역공학 방지 처리.
